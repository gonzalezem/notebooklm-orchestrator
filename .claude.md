# Claude Instructions for this Repo

## Purpose
This repo builds a CLI-first pipeline that curates sources (YouTube + URLs), pushes them into NotebookLM, requests analysis + deliverables, and downloads artifacts locally with a reproducible manifest.

## Non-negotiables
- Spec-driven workflow only: interview -> SPEC.md -> plan -> implement. Do not skip steps.
- Do not change top-level folder structure without explicit instruction.
- All runtime outputs go under `outputs/<run_id>/` and must include `run_manifest.json`.
- Never commit secrets, cookies, auth caches, or browser profiles. Add to `.gitignore` if created.
- Prefer minimal dependencies. If you want to add one, justify it and update `pyproject.toml`.

## Repo layout (do not invent new top-level dirs)
- `src/` python package
- `scripts/` helper scripts
- `prompts/` prompt templates
- `configs/` defaults/config
- `tests/`
- `outputs/` (gitignored)

## Commands
- Setup: `python3 -m venv .venv && source .venv/bin/activate && pip install -U pip && pip install -e .`
- Tests: `pytest -q`
- Lint (if configured): `ruff check .`

## Definition of done for a change
- Code compiles/runs locally on macOS.
- Tests updated or added for logic changes.
- Manifest/provenance preserved.
- README updated if the user-facing CLI changed.

## When unsure
Ask me. Do not guess about credentials, notebook ids, or external tool behavior.
