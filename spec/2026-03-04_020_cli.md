# CLI Spec: nlm-orch
Date: 2026-03-04
Status: v1 (post-interview 2026-03-05)

---

## Command name (locked)
`nlm-orch`

---

## Subcommands (v1)

```
nlm-orch sources <query> [flags]   # curate sources only
nlm-orch run <query> [flags]       # full pipeline
nlm-orch login                     # NotebookLM auth
nlm-orch doctor                    # environment check
nlm-orch --version
nlm-orch --help
```

`<query>` is a required positional argument for `sources` and `run`.
`--config <path>` is accepted by both and wins over all other flags on conflict.

---

## Exit codes (uniform across all subcommands)

| Code | Meaning |
|---|---|
| 0 | Success |
| 1 | Runtime error (unexpected failure) |
| 2 | Invalid arguments |
| 3 | No sources found after curation |
| 4 | Required tool missing (`yt-dlp` or `notebooklm`) |
| 5 | Auth failure (NotebookLM login required) |

---

## `nlm-orch sources <query>`

**Purpose:** Collect YouTube candidates via yt-dlp, apply curation filters, write `raw.jsonl` and `sources.json`.

**Positional:**
- `<query>` (required) -- search query string

**Flags:**
| Flag | Default | Description |
|---|---|---|
| `--config PATH` | none | Config file (YAML/TOML). Wins over all flags. |
| `--max-results INT` | 50 | Max YouTube results to fetch before curation. |
| `--recency WINDOW` | `6months` | Recency window. Values: `Nd`, `Nmonths`, `Nyears`, `all`. |
| `--max-duration DURATION` | `30m` | Max video duration. Values: `Nm` or `Nh`. |
| `--min-views INT` | 1000 | Min view count. |
| `--channel-allow TEXT` | none | Comma-separated channel allowlist. |
| `--channel-block TEXT` | none | Comma-separated channel blocklist. |
| `--run-id TEXT` | auto | Override auto-generated run ID. |
| `--outputs-dir PATH` | `outputs` | Root outputs directory. |

**Outputs written:**
```
outputs/<run_id>/
  raw.jsonl       # all yt-dlp metadata before curation
  sources.json    # curated list (included=true sources only)
  run_manifest.json
  run.log
```

**Exit on `--max-results` exceeded cap:** cap at 50 silently, emit a warning to `run.log`.

---

## `nlm-orch run <query>`

**Purpose:** Full pipeline: curate sources, create NotebookLM notebook, upload sources, run prompts, generate deliverables, download artifacts, write manifest.

**Positional:**
- `<query>` (required unless `--sources` is given)

**Flags (inherits all `sources` flags, plus):**
| Flag | Default | Description |
|---|---|---|
| `--sources PATH` | none | Use a pre-existing `sources.json` instead of running curation. Skips yt-dlp. |
| `--notebook-id TEXT` | none | Reuse an existing NotebookLM notebook by ID. Default: create new notebook each run. |
| `--prompts PATH` | none | Repeatable. Each file is a plain `.txt` or `.md` prompt sent to NotebookLM in order. |
| `--deliverables LIST` | `slides infographic briefing` | Space-separated list. Valid values: `slides`, `infographic`, `briefing`. |
| `--intent TEXT` | `strategy` | Prompt pack to load from `prompts/packs/<intent>/`. Valid values: `strategy`, `implementation`, `deliverables`. |
| `--dry-run` | off | Stop after curation. Write `sources.json` and manifest with `status=dry-run`. Do not touch NotebookLM. |

**Deliverable keyword-to-filename map:**
```
slides      -> artifacts/deck.pdf
infographic -> artifacts/infographic.png
briefing    -> artifacts/briefing.md
```

**Outputs written:**
```
outputs/<run_id>/
  raw.jsonl
  sources.json
  run_manifest.json
  run.log
  artifacts/
    deck.pdf          # if "slides" in --deliverables
    infographic.png   # if "infographic" in --deliverables
    briefing.md       # if "briefing" in --deliverables
```

## `nlm-orch login`

**Purpose:** Perform NotebookLM auth once per machine. Delegates to the NotebookLM Python client.

**Behavior:**
- Calls `notebooklm login` subprocess.
- On success: prints confirmation and path to `~/.notebooklm/storage_state.json`.
- On failure: prints actionable error and exits 5.

**No flags.**

---

## `nlm-orch doctor`

**Purpose:** Validate environment and required tool paths.

**Checks (in order):**
1. Python version >= 3.11
2. `yt-dlp` path (prefer venv-local, then Homebrew, then PATH)
3. `notebooklm` CLI path (same resolution order)
4. `~/.notebooklm/storage_state.json` exists (auth state present)

**Output:** JSON to stdout. One warning line to stderr per missing item.

**Exit (uniform exit code table):**
- 4 if `yt-dlp` or `notebooklm` CLI is missing
- 5 if both tools are found but `~/.notebooklm/storage_state.json` is absent
- 0 if all checks pass

## Run ID generation

Format: `<slug>_<YYYYMMDD_HHMMSS>` (local time)

Slug rules (applied in order):
1. Lowercase the query string.
2. Replace any character that is not `[a-z0-9]` with `_`.
3. Collapse consecutive underscores into one.
4. Strip leading and trailing underscores.
5. Truncate to 40 characters.

Example: `microbiome_cancer_20260304_143022`

If `--run-id` is provided, use it verbatim without slug processing.

---

## Logging

- `run.log` is always written inside the run folder.
- Console output is short and actionable (run folder path, status, errors).
- Verbose detail goes to `run.log` only.
- Warnings (e.g. cap exceeded, tool not found) go to stderr and to `run.log`.

---

## Config file format

YAML or TOML. Keys match long flag names with hyphens replaced by underscores.

Example YAML:
```yaml
max_results: 30
recency: 3months
max_duration: 20m
min_views: 500
deliverables:
  - slides
  - briefing
prompts:
  - prompts/overview.md
  - prompts/gaps.md
```

Config file wins over CLI flags on all keys present. Missing keys fall through to flag defaults.

## Conflicts with seed spec (resolved)

| Topic | Seed CLI spec | Decision |
|---|---|---|
| Run ID order | `YYYYMMDD_HHMMSS_<slug>` | Changed to `<slug>_YYYYMMDD_HHMMSS` (core spec wins) |
| Notebook flag | `--notebook NAME` with reuse-by-name | Changed to `--notebook-id` only; no reuse-by-name in v1 |
| Query input | `--query TEXT` named flag | Changed to positional `<query>` (core spec wins) |
| Flag names | `--count`, `--days`, `--max-duration-sec` | Changed to `--max-results`, `--recency`, `--max-duration` |
| `--resume` | Listed in draft | Deferred past v1 |
| `--dry-run` | Listed in draft | Kept in v1 |

Note: `cli.py` currently implements the old flag names and order. It must be updated in Phase 3 or earlier.

## Testing plan

- Unit: `_slugify` edge cases (special chars, truncation, collapse).
- Unit: run ID generation with and without `--run-id` override.
- Unit: `--deliverables` parsing and keyword-to-filename mapping.
- Unit: config file loading and flag precedence (config wins over flag, flag wins over default).
- Unit: exit code correctness for each failure mode.
- Integration: `nlm-orch doctor` exits 0 with `yt-dlp` present.
- Integration: `nlm-orch sources "test query"` writes `sources.json` and `run_manifest.json` under `outputs/`.
- Integration: `nlm-orch run "test query" --dry-run` exits 0 and writes `status=dry-run` manifest without touching NotebookLM.
