# Source Spec: YouTube (yt-dlp)
Date: 2026-03-04
Status: v1 (post-interview 2026-03-05)

---

## Purpose
Provide deterministic YouTube source collection and curation to feed NotebookLM with up to 50 high-signal sources per run.

---

## Goals
1. Execute a yt-dlp metadata search against YouTube for a given query.
2. Write all raw results to `raw.jsonl` before any filtering.
3. Apply curation filters in a defined order and produce `sources.json` with inclusion decisions recorded for every candidate.
4. Fail fast with the correct exit code and a human-readable error message.

## Non-goals
- Downloading audio or video files.
- Searching sources other than YouTube (URLs and local files are handled by a separate future module).
- Exclude-terms filtering (deferred to Phase 3).
- Channel-based playlist or subscription traversal.

---

## Constraints
- yt-dlp is called as a subprocess. Python API binding is not used in v1.
- NotebookLM accepts at most 50 sources per notebook. The curator must cap output at 50.
- All filter defaults are from the locked core spec and CLI spec. Seed spec values are superseded (see Decision log).

---

## yt-dlp invocation

```bash
yt-dlp --dump-json --flat-playlist "ytsearch<N>:<query>"
```

- `N` = value of `--max-results` (default 50, hard cap 50).
- `--dump-json` emits one JSON object per line (NDJSON), one per video.
- `--flat-playlist` avoids resolving individual video pages for search results; uses search metadata only.
- stdout is captured line by line and written to `raw.jsonl`.
- stderr is captured and written to `run.log`.

If `--max-results` exceeds 50, cap silently at 50 and emit a warning to `run.log`.

---

## Retry policy
- If yt-dlp exits with a non-zero code, wait 3 seconds and retry once with the identical command.
- If the second attempt also fails: write `status=failed` and `error_summary` to `run_manifest.json`, then exit 1.
- `raw.jsonl` may be partially written on failure; leave it in place for debugging.

## Filtering pipeline (applied in order after raw.jsonl is written)

All filters use the values passed via flags or config. Defaults from core spec:

| Filter | Default | CLI flag |
|---|---|---|
| Recency | 6 months | `--recency` |
| Max duration | 30 minutes | `--max-duration` |
| Min views | 1000 | `--min-views` |
| Channel allowlist | none | `--channel-allow` |
| Channel blocklist | none | `--channel-block` |

### Filter value parsing

**Recency:** parse `--recency` value as:
- `Nd` = N days ago
- `Nmonths` = N * 30 days ago
- `Nyears` = N * 365 days ago
- `all` = no recency filter

**Max duration:** parse `--max-duration` value as:
- `Nm` = N minutes (in seconds: N * 60)
- `Nh` = N hours (in seconds: N * 3600)

**Channel allow/block:** comma-separated strings matched case-insensitively against the `channel` field.

### Filter order
1. Recency: drop if `published_at` is older than the window. Keep if `published_at` is unknown.
2. Max duration: drop if `duration_seconds` exceeds the limit. Keep if `duration_seconds` is unknown.
3. Min views: drop if `view_count` is below the minimum. Keep if `view_count` is unknown.
4. Channel blocklist: drop if `channel` matches any blocked channel.
5. Channel allowlist: if non-empty, drop if `channel` does not match any allowed channel.
6. Deduplicate by video ID (see Dedupe rules below).
7. Cap at 50: if more than 50 sources remain, keep the 50 with the highest `view_count`.

Every candidate from `raw.jsonl` appears in `sources.json` with `included=true` or `included=false` and an `exclusion_reason` string if excluded.

---

## Dedupe rules
- Primary key: YouTube video ID (extracted from URL or `id` field in yt-dlp output).
- On duplicate video ID: keep the entry with the most recent `published_at`. If `published_at` is equal or missing, keep the entry with the higher `view_count`.

## Output files

### `raw.jsonl`
One yt-dlp JSON object per line, exactly as returned by `--dump-json`. No transformation.
Written before any filtering begins.

### `sources.json`
Array of objects. One entry per candidate from `raw.jsonl` (including excluded ones).

```json
[
  {
    "type": "youtube",
    "url": "https://www.youtube.com/watch?v=<id>",
    "title": "...",
    "channel": "...",
    "published_at": "YYYY-MM-DD",
    "duration_seconds": 0,
    "view_count": 0,
    "included": true,
    "exclusion_reason": null
  }
]
```

Field notes:
- `type`: always `"youtube"` for entries produced by this module.
- `published_at`: ISO date string (`YYYY-MM-DD`). Null if yt-dlp does not return it.
- `duration_seconds`: integer. Null if unknown.
- `view_count`: integer. Null if unknown.
- `included`: `true` if the source passed all filters and is within the 50-cap. `false` otherwise.
- `exclusion_reason`: null if `included=true`. Otherwise a short string: `"recency"`, `"max_duration"`, `"min_views"`, `"channel_block"`, `"channel_allow"`, `"cap"`, or `"duplicate"`.

**Note:** adding `type` to this schema requires updating the `sources.json` field definition in `spec/2026-03-04_010_core.md`. See Decision log.

## Exit codes (from uniform table in `020_cli.md`)

| Code | Condition |
|---|---|
| 0 | Success: at least one source included |
| 1 | Runtime error: yt-dlp failed after retry |
| 2 | Invalid arguments |
| 3 | No sources remain after all filters applied |
| 4 | yt-dlp not found |

---

## Failure modes

| Failure | Behavior |
|---|---|
| yt-dlp not on PATH or venv | Exit 4. Print: "yt-dlp not found. Install with: brew install yt-dlp" |
| yt-dlp exits non-zero (network, quota) | Retry once after 3s. On second failure: exit 1, write error_summary to manifest. |
| Zero sources after filtering | Exit 3. Print: "No sources matched filters. Try relaxing --recency, --min-views, or --max-duration." |
| `published_at` missing for a candidate | Treat as passing the recency filter (unknown is not excluded). Log to run.log. |
| `duration_seconds` missing | Treat as passing the max-duration filter. Log to run.log. |
| `view_count` missing | Treat as passing the min-views filter. Log to run.log. |

## Testing plan
- Unit: recency window parsing (`6months`, `90d`, `2years`, `all`).
- Unit: duration parsing (`30m`, `1h`, `90m`).
- Unit: filter pipeline with a fixture of 10 candidate objects; verify included/excluded counts and exclusion_reasons.
- Unit: dedupe by video ID (newer wins; higher views wins on tie).
- Unit: 50-source cap (input 60 candidates, verify output has 50 with highest view_counts included).
- Unit: unknown field handling (null published_at/duration_seconds/view_count pass their respective filters).
- Integration: mocked yt-dlp subprocess returning fixture NDJSON; verify raw.jsonl written, sources.json written, manifest updated.
- Integration: mocked yt-dlp exits 1 on first call, 0 on second; verify retry succeeds and run exits 0.
- Integration: mocked yt-dlp exits 1 twice; verify exit 1 and error_summary in manifest.
- Integration: filters produce zero results; verify exit 3.
