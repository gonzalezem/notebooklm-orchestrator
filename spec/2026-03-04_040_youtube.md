# Source Spec: YouTube (yt-dlp)
Date: 2026-03-04
Status: draft

## Purpose
Provide deterministic YouTube sourcing + filtering to feed NotebookLM with <=50 high-signal sources.

## Implementation constraints
- Use yt-dlp (subprocess call is acceptable in v1).
- Store raw yt-dlp jsonl output in outputs/<run_id>/raw_ytdlp.jsonl.

## Filtering policy (defaults, v1)
- count: 20 (cap 50)
- recency: 90 days
- min views: 5000
- max duration: 3600 sec
- exclude terms: ["reaction", "livestream", "podcast"] (editable via config)

## Dedupe rules
- Primary key: video id
- If duplicates, keep the newest (publish date) or highest views if publish date missing.

## Output schema: sources.json
Array of objects:
- type: "youtube"
- url
- title
- channel
- views
- duration_sec
- publish_date (ISO date if known)
- selected_reason (string, optional but useful)

## Failure modes
- yt-dlp missing -> exit code 4 with install hint
- empty results after filtering -> exit code 3 with suggestion to relax filters
- intermittent network -> retry yt-dlp once
