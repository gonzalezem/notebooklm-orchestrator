# Testing and Ops Spec
Date: 2026-03-04
Status: v1 (post-interview 2026-03-05)
Owner: Emmanuel Gonzalez

---

## Goals
1. Every code change is validated by an automated test suite before merging.
2. Test coverage is measured and reported in CI (not gated).
3. Live end-to-end tests can be run locally against real NotebookLM but are excluded from CI.
4. Ops concerns (log location, cleanup) are documented so the repo stays tidy.

## Non-goals
- Background services, daemons, or scheduled runs in v1.
- Alerting or external monitoring.
- Automated output directory cleanup (documented shell one-liner only).
- Dedicated staging or production environment.

---

## Test categories

### 1. Unit tests
Pure logic, no subprocess calls, no filesystem side effects beyond tmp_path.

Coverage targets:
- `_slugify` / `_make_run_id`: slug rules, truncation, timestamp format.
- `_version_ok`: equal, greater, below, unparseable, empty.
- `parse_recency` / `parse_duration`: all documented formats and edge cases.
- `apply_filters`: each filter stage independently; null field exclusion; cap; dedupe.
- `normalize_entry`: field mapping, upload_date conversion, fallback keys.

Files: `tests/test_sources.py`, `tests/test_run.py` (version_ok tests).

### 2. Mocked integration tests
Subprocess calls (`notebooklm`, `yt-dlp`) are monkeypatched. Real filesystem under `tmp_path`.

Coverage targets:
- `cmd_run` happy path: all fields in manifest, all artifact files created, notes/ written.
- `cmd_run` partial paths: zero sources, all sources failed, deliverable download failure.
- `cmd_run` abort paths: notebooklm missing (exit 4), auth missing (exit 5), version gate (exit 4).
- `cmd_run` warning paths: source add failure with at least 1 success, wait timeout, no source_id.
- `cmd_run` notebook reuse: `--notebook-id` skips `create_notebook`.
- `cmd_run` dry-run: no NLM calls, `artifacts/` and `notes/` dirs created, manifest `status=dry-run`.
- `cmd_sources`: curation result written to `raw.jsonl` and `sources.json`, manifest correct.
- Rate limiting: `time.sleep(2)` called N-1 times for N sources.

Files: `tests/test_run.py`, `tests/test_sources.py`.

### 3. Live tests (local only, skipped in CI)
Hit real NotebookLM and real `yt-dlp`. Require auth state at `~/.notebooklm/storage_state.json`.

Marker: `@pytest.mark.live`

Invocation:
```bash
pytest -m live
```

Scope: one test matching the SOP smoke test.
- Query: `"claude code skills"`, `--deliverables briefing`.
- Assertions: `artifacts/briefing.md` exists, `notes/ask_0.md` exists, `run_manifest.json` has `status` in `{"success", "partial"}` with no `failed_step`.

CI: live tests are **excluded** via `addopts = "-m 'not live'"` in `pyproject.toml`.

File: `tests/test_live.py`

---

## Pytest configuration (pyproject.toml)

Add to `pyproject.toml`:

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-m 'not live'"
markers = [
    "live: requires real NotebookLM auth and network access (skipped in CI)",
]
```

This eliminates `PytestUnknownMarkWarning` and ensures live tests never run in CI.

## Coverage

Add `pytest-cov` to dev dependencies:

```toml
[project.optional-dependencies]
dev = ["pytest>=8.0", "pytest-cov"]
```

Run with:
```bash
pytest --cov=notebooklm_orchestrator --cov-report=term-missing
```

Coverage is **reported but not gated**. CI prints the report; a drop in coverage is visible but does not fail the build.

## CI (GitHub Actions)

Trigger: push to `main`, pull requests to `main`.

Runner: `ubuntu-latest`.

Matrix: Python `3.10` and `3.11`.

Steps:
1. Checkout repo.
2. Set up Python (matrix version).
3. `pip install -e ".[dev]"`.
4. `pytest --cov=notebooklm_orchestrator --cov-report=term-missing`.
   Live tests excluded automatically via `addopts`.

Workflow file: `.github/workflows/ci.yml`

No secrets, no external service credentials in CI. All NLM and yt-dlp calls are mocked.

## Ops

### Logs
- Location: `outputs/<run_id>/run.log`
- Content: every subprocess command (`CMD: ...`), stdout/stderr of each call, warnings, errors.
- No log rotation in v1. Old runs accumulate in `outputs/`.

### Output directory cleanup
No automated cleanup. To remove old runs manually:

```bash
# Keep the 5 most recent; delete the rest
ls -dt outputs/*/ | tail -n +6 | xargs rm -rf
```

Document this one-liner in `docs/SOP.md` under a "Cleanup" subsection.

### No background services
- No daemons, no schedulers, no persistent processes.
- Every `nlm-orch` invocation is a self-contained subprocess.
