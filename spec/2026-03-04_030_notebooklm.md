# Integration Spec: NotebookLM
Date: 2026-03-04
Status: v1 (updated 2026-03-05)

## Purpose
Automate NotebookLM as the analysis + deliverables engine while keeping the run locally auditable.

## Integration surface (v1)
- Use notebooklm CLI (unofficial) as the first implementation.
- Treat it as brittle: clear errors, no silent fallback.

## Notebook strategy (locked)
- Default: create a new notebook per run. Name equals run_id.
- `--notebook-id <id>`: reuse an existing notebook by ID. No reuse-by-name in v1.
- Record notebook name + id in run_manifest.json.
- Notebook creation or selection failure → `status=partial`, early exit.

## Operations (v1)
1) Create or select notebook
2) Add sources from sources.json
3) Ask analysis prompts (from prompts/ files)
4) Generate deliverables:
   - slides (deck.pdf)
   - infographic (infographic.png)
   - briefing (briefing.md or txt)
5) Download artifacts into outputs/<run_id>/artifacts/

## Prompt handling
- Prompts are versioned files in repo (prompts/)
- run_manifest.json records prompt file paths and the exact text used

## Source add behavior
- Add sources one at a time, waiting for each to process before the next.
- Individual add failures: skip and continue. Record failed URL in `sources_failed_urls` and in `warnings[]` as `type: source_add_failed`.
- If ALL sources fail to add: `status=partial`, abort before prompts/deliverables.
- If at least 1 source added and all deliverables succeed: `status=success` regardless of how many sources failed.

## Status semantics (locked)
`status=partial` only when:
- notebook creation/selection failed, OR
- zero sources were added successfully, OR
- at least one requested deliverable is missing or failed.

Non-fatal (warnings only, not partial):
- Some source add operations fail (at least 1 succeeds).
- A prompt ask fails.

## Failure modes
- Auth expired: tell user to run `nlm-orch login`. Exit 5.
- notebooklm CLI not found: exit 4.
- Source limit exceeded: curation caps at 20 selected; NLM hard cap is 50. Pruning happens before upload.
- Video unavailable: add_source returns ok=False; recorded as warning.
- NotebookLM client/API break: fail fast, write partial manifest, print error.
