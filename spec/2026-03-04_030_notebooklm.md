# Integration Spec: NotebookLM
Date: 2026-03-04
Status: v1 (post-interview 2026-03-05)
Owner: Emmanuel Gonzalez

---

## Purpose
Automate NotebookLM as the analysis and deliverables engine for `nlm-orch run`. All operations go through the `notebooklm` subprocess CLI (unofficial client). Treat it as brittle: fail fast, clear errors, no silent fallback.

---

## Goals
1. Create or reuse a NotebookLM notebook per run.
2. Add curated YouTube source URLs to the notebook.
3. Ask analysis prompts and save responses to `notes/`.
4. Generate and download deliverables (slides, infographic, briefing) into `artifacts/`.
5. Log every CLI command executed and record provenance in `run_manifest.json`.

## Non-goals
- Local file upload (deferred past v1).
- Parallel artifact generation (deferred; sequential is safer).
- Notebook deduplication or name-based lookup.
- Automatic notebook deletion on failure.

---

## Minimum CLI version
- **Minimum: `0.3.3`** (tested baseline).
- Version is checked during preflight (`notebooklm --version`). If the parsed version is below `0.3.3`, fail fast with exit 4 and a clear error message.
- Version string is recorded in `run_manifest.json` as `notebooklm_version` for auditability.
- If version string cannot be parsed, log a warning and continue (do not fail).

---

## CLI executable resolution
1. Check `sys.prefix/bin/notebooklm` (venv-local, preferred).
2. Fall back to `shutil.which("notebooklm")` (PATH).
3. If not found: exit 4.

## Auth state
- Expected at: `~/.notebooklm/storage_state.json`
- Checked during preflight. If absent: exit 5, tell user to run `nlm-orch login`.
- Never committed. Listed in `.gitignore`.

---

## Notebook strategy (locked)
- **Default**: create a new notebook per run. Name = `run_id`.
- **`--notebook-id <id>`**: reuse an existing notebook by ID. `create_notebook` is NOT called; `use_notebook` is called instead.
- No reuse-by-name in v1.
- Record `notebook_id` and `notebook_name` in `run_manifest.json`.
- Notebook creation or `use_notebook` failure → `status=partial`, `failed_step=notebook`, early exit (return 1).
- Notebooks created during partial/failed runs are NOT deleted. They are left in NotebookLM for manual inspection.

## Operations and CLI commands

### 1. Create notebook
```
notebooklm create <name> --json
```
Response: `{"notebook": {"id": "...", "title": "..."}}`
Raises `RuntimeError` on non-zero exit or missing `id` field.

### 2. Use notebook
```
notebooklm use <notebook_id>
```
Raises `RuntimeError` on non-zero exit.

### 3. Add source
```
notebooklm source add <url> --json -n <notebook_id>
```
Response: `{"source": {"id": "...", ...}}`
Never raises. Returns `{"ok": bool, "source_id": str|None, "error": str|None}`.

Failure handling:
- Non-zero exit → `ok=False`, record error in `warnings[]` as `type: source_add_failed`.
- `ok=True` but `source_id` is `None` (JSON parse succeeded but id missing) → count as succeeded, add warning `type: source_add_no_id`.
- Timeout during `_run` → treated as `ok=False`.

### 4. Wait for source
```
notebooklm source wait <source_id> -n <notebook_id> --timeout <seconds>
```
Timeout: 90 seconds (outer subprocess timeout: 105 seconds).

Wait failure handling:
- Returns `False` (timeout or non-zero exit) → log warning `type: source_wait_timeout`, **count the source as succeeded** (it was accepted; wait failure is non-fatal).
- If `source_id` is `None` (from step 3 edge case) → skip `wait_source` call entirely.

**Rate limiting**: sleep 2 seconds after each `wait_source` call (success or failure) before adding the next source. This applies to every source except the last.

### 5. All-sources-failed guard
After all sources are processed: if `add_ok == 0` and `len(urls) > 0` → `status=partial`, `failed_step=add_sources`, early exit (return 1).

### 6. Ask prompt
```
notebooklm ask "<prompt_text>" --json -n <notebook_id>
```
Timeout: 120 seconds.
Response: `{"answer": "..."}` (fall back to raw stdout if JSON parse fails).
Raises `RuntimeError` on non-zero exit. Caller catches and logs; recorded as `type: prompt_failed` in `warnings[]`. Non-fatal.

### 7. Generate artifact
```
notebooklm generate <gen_type> --json --no-wait -n <notebook_id>
```
`gen_type` values: `report`, `slide-deck`, `infographic`
Response: `{"task_id": "..."}` (also accept `"id"` key as fallback).
Timeout: 60 seconds.
Raises `RuntimeError` on non-zero exit or missing task_id.

### 8. Wait for artifact
```
notebooklm artifact wait <task_id> -n <notebook_id> --timeout <seconds> --json
```
Timeout: 360 seconds (outer subprocess timeout: 390 seconds).
Returns `True` on success, `False` on timeout/failure.

### 9. Download artifact
```
notebooklm download <dl_type> <dest_path> --force -n <notebook_id>
```
`dl_type` values: `report`, `slide-deck`, `infographic`
Timeout: 120 seconds.
Returns `True` if exit 0 AND dest_path exists on disk.

## Deliverable keyword map (locked)

| Keyword | gen_type | dl_type | filename |
|---|---|---|---|
| `briefing` | `report` | `report` | `briefing.md` |
| `slides` | `slide-deck` | `slide-deck` | `deck.pdf` |
| `infographic` | `infographic` | `infographic` | `infographic.png` |

Generation is **sequential**: generate → wait → download for each deliverable in order, before starting the next.

---

## Prompt handling
- Prompt files live in `prompts/` and are versioned in the repo.
- Default: `prompts/analysis.md` (used if no `--prompts` flags given and file exists).
- Multiple prompts run in the order given on the CLI.
- Each prompt response is saved to `outputs/<run_id>/notes/ask_<N>.md` (N = 0-indexed order).
- `run_manifest.json` records under `prompts_used[]`: `file`, `text` (exact content used), `response_file` (absolute path, or `null` if ask failed).

---

## Outputs contract (including notes/)

```
outputs/<run_id>/
  raw.jsonl
  sources.json
  run_manifest.json
  run.log
  artifacts/          # always created (empty on --dry-run)
    deck.pdf
    infographic.png
    briefing.md
  notes/              # created at run-dir init; may be empty if no prompts run
    ask_0.md
    ask_1.md
    ...
```

`notes/` is part of the locked outputs contract. It is created at run-dir initialization alongside `artifacts/`.

---

## Status semantics (locked)

| Status | Condition |
|---|---|
| `success` | Notebook created/selected, ≥1 source added, all requested deliverables downloaded. |
| `partial` | Notebook failed, OR zero sources added, OR ≥1 requested deliverable missing/failed. |
| `failed` | Hard abort: tool missing, auth failure, yt-dlp error. `failed_step` and `error_summary` set. |
| `dry-run` | Stopped after curation. No NLM calls made. |

**Non-fatal (warnings only)**:
- Some source adds fail (at least 1 succeeds): `warnings[]` entry `type: source_add_failed`.
- `wait_source` times out: `warnings[]` entry `type: source_wait_timeout`. Source counted as succeeded.
- `add_source` ok=True but `source_id` absent: `warnings[]` entry `type: source_add_no_id`.
- A prompt ask fails: `warnings[]` entry `type: prompt_failed` with `prompt_index`.

## Logging
- Every subprocess call logs `CMD: <full command>` to `run.log` before execution.
- Stdout and stderr of each call appended to `run.log`.
- All warnings and errors appended to `run.log` with descriptive messages.

## Timeouts (all values in seconds)

| Operation | subprocess timeout | outer timeout |
|---|---|---|
| `--version` | 15 | 15 |
| `create`, `use` | 120 | 120 |
| `source add` | 120 | 120 |
| `source wait` | 90 | 105 |
| `ask` | 120 | 120 |
| `generate` | 60 | 60 |
| `artifact wait` | 360 | 390 |
| `download` | 120 | 120 |

---

## Failure modes

| Condition | Exit code | Status | Action |
|---|---|---|---|
| `notebooklm` not found | 4 | `failed` | Print install instructions |
| Version < 0.3.3 | 4 | `failed` | Print version mismatch message |
| Auth state absent | 5 | `failed` | Tell user to run `nlm-orch login` |
| Notebook create/use fails | 1 | `partial` | Print error, write manifest |
| All sources fail to add | 1 | `partial` | Print error, write manifest |
| Individual source fails | 0 | warning | Record in `warnings[]`, continue |
| Artifact generation fails | 0 | `partial` | Record in `missing_artifacts[]`, continue |
| NLM CLI API break | 1 | `partial` or `failed` | Fail fast, write manifest, print error |
