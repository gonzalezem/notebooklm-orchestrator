# Spec: Curation Review Mode
Date: 2026-03-07
Status: v1 (post-interview 2026-03-07)

---

## Goals

1. Let the user inspect and edit curated sources before committing to a full NotebookLM run.
2. Write a structured `curation_report.md` after curation: included sources table, excluded summary by reason, and rerun instructions.
3. Stop before all NotebookLM operations (no notebook create, no source upload, no deliverables).
4. Leave `sources.json` in an editable state so the user can reinstate or remove sources and rerun with `--sources`.

## Non-goals

- No changes to NotebookLM operations, `nlm-orch sources`, `nlm-orch doctor`, or `nlm-orch login`.
- No verbose excluded-source listing (summary counts only; no `--review-verbose` flag).
- No interactive editor or TUI.
- Not a replacement for `--dry-run` (different intent: curation inspection vs pipeline gate).

---

## CLI change

One new flag on `nlm-orch run`:

```
--review    Stop after curation and write curation_report.md. No NotebookLM calls.
```

- Type: boolean, default off.
- Compatible with all curation flags (`--max-results`, `--recency`, `--max-duration`, `--min-views`, `--channel-allow`, `--channel-block`).
- Compatible with `--sources` (see behavior below).
- `--review` + `--dry-run` together: `--review` wins silently. `--dry-run` flag is ignored.
- No new flags beyond `--review`.

### Behavior when `--sources` is passed

If `--sources <path>` is passed alongside `--review`, yt-dlp is skipped (existing behavior). The report is generated from the provided `sources.json`. This allows re-inspecting a prior run's source set before rerunning with edits.

---

## Outputs contract addition

When `--review` is active:

```
outputs/<run_id>/
  raw.jsonl             # written if yt-dlp ran; absent if --sources was used
  sources.json          # always written (or copied from --sources input)
  run_manifest.json     # status="review"
  run.log               # always written
  curation_report.md    # NEW: always written on --review
  artifacts/            # created empty (existing contract)
  notes/                # created empty (existing contract)
```

`curation_report.md` is not written on normal runs or `--dry-run` runs.

---

## `curation_report.md` format

A plain Markdown file generated by the CLI from `sources.json`. Three sections, always in this order.

### Section 1: Included sources

Heading: `## Included sources (N)`

Markdown table:

| # | Title | Channel | Views | Duration | Published | URL | Notes |
|---|---|---|---|---|---|---|---|

- `#`: 1-based index matching position in `sources.json` (for editing reference).
- `Views`: raw integer from `view_count`.
- `Duration`: formatted as `Xm Ys` (e.g., `12m 30s`). Source field: `duration_seconds`.
- `Published`: `published_at` formatted as `YYYY-MM-DD`.
- `Notes`: empty string. Placeholder for manual annotation.
- One row per source where `included=true`.

### Section 2: Excluded sources summary

Heading: `## Excluded sources (N)`

Breakdown table by `exclusion_reason`. Include a row for every reason present in the data. `cap` is its own row (sources that passed all filters but were dropped by `SELECTION_CAP=20`).

| Exclusion reason | Count |
|---|---|
| channel_allow | N |
| channel_block | N |
| recency | N |
| max_duration | N |
| min_views | N |
| duplicate | N |
| cap | N |

Rows with count 0 are omitted. Rows appear in the order listed above (canonical reason order).

### Section 3: How to edit and rerun

Heading: `## How to edit sources.json and rerun`

Fixed prose (not generated from data):

```
To edit the curated source set before running the full pipeline:

1. Open outputs/<run_id>/sources.json
2. To remove an included source: set "included": false on that entry.
3. To reinstate an excluded source: set "included": true and set "exclusion_reason": null.
4. Rerun with:

   nlm-orch run "<original query>" --sources outputs/<run_id>/sources.json [other flags]

   Omit --review on the rerun to proceed to NotebookLM.
```

The run_id and query are interpolated from the current run.

## `run_manifest.json` changes

When `--review` is active, the manifest contains:

```json
{
  "status": "review",
  "review_report_path": "<absolute path to curation_report.md>",
  ...
}
```

- `status`: `"review"` (new value; existing: `success`, `partial`, `failed`, `dry-run`).
- `review_report_path`: absolute path string.
- Fields absent in review mode (never populated): `notebook_id`, `prompts`, `artifacts`.
- All curation fields present as normal: `inputs`, `sources_file`, `included_count`, `excluded_count`, `warnings`.

---

## stdout behavior

```
DONE: <run_dir> [status=review sources=N/M report=curation_report.md]
```

- `N` = included count, `M` = total candidates.
- `report=curation_report.md` is the filename only (not the full path; run_dir is already shown).
- No `artifacts=` counter (none attempted in review mode).

---

## Exit codes

| Code | Condition |
|---|---|
| 0 | Curation complete, `curation_report.md` written |
| 1 | Unexpected runtime error |
| 2 | Invalid arguments (not triggered by `--review` + `--dry-run`; review wins silently) |
| 3 | No included sources after curation; `curation_report.md` not written |

Exit 3 behavior: consistent with existing no-sources handling. Manifest written with `status=failed`, `failed_step=sources`. No report file created.

---

## Acceptance criteria

1. `nlm-orch run "<query>" --review` exits 0; `curation_report.md` is written; no `nl_cli.create_notebook` call made.
2. `run_manifest.json` has `status="review"` and `review_report_path` set to the absolute path of the report.
3. `curation_report.md` contains all three sections: `## Included sources`, `## Excluded sources`, `## How to edit sources.json and rerun`.
4. Included table row count equals the count of `included=true` entries in `sources.json`.
5. Excluded breakdown counts sum to the total number of `included=false` entries in `sources.json`. `cap` appears as its own row when present.
6. `Duration` column is formatted as `Xm Ys`, not raw seconds.
7. `Notes` column is present and empty in every included row.
8. `artifacts/` and `notes/` directories exist and are empty.
9. `DONE:` line printed to stdout with `status=review sources=N/M report=curation_report.md`.
10. `--review` + `--dry-run` together: `--review` wins; report is written; exit 0.
11. `--review` + `--sources <path>`: curation skipped; report generated from provided `sources.json`; exit 0.
12. Zero included sources: exit 3; `curation_report.md` not written; manifest `status=failed`.
