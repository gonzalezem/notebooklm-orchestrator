# Core Spec: NotebookLM Orchestrator
Date: 2026-03-04
Status: v1 (post-interview)
Owner: Emmanuel Gonzalez

## One-sentence pitch
CLI-first research/report pipeline that curates sources, pushes them into NotebookLM, requests analyses and deliverables, and downloads artifacts into a local run folder with a reproducible evidence trail.

---

## Goals (must be true when we ship)
1. `nlm-orch sources <query>` collects and curates YouTube sources; writes `raw.jsonl` and `sources.json`.
2. `nlm-orch run <query>` executes the full pipeline end-to-end: collect, curate, create notebook, upload sources, generate deliverables, download artifacts.
3. Every run produces an auditable `run_manifest.json`: query, filters applied, prompts used, notebook id, artifact paths, timestamps, status, error summary if any.
4. Works reliably on macOS with Python 3.11.

## Non-goals
- Custom RAG backend or vector DB.
- Paywall bypass or access-control circumvention.
- GUI beyond CLI and NotebookLM UI.
- Topic-based notebook lookup table (deferred past v1).

---

## Hard constraints
- Platform: macOS, Python 3.11.
- NotebookLM source cap: 50 sources per notebook.
- Auth: interactive Google login at least once; no secrets committed.
- NotebookLM automation is unofficial and may break on Google internals changes. Fail fast with a clear error message and partial manifest.

---

## CLI interface

### Primary input
- Positional argument: `<query>` string (required for ad-hoc use).
- `--config <path>`: YAML or TOML config file. Config wins on all conflicts with flags.

### Subcommands
```
nlm-orch sources <query> [flags]   # collect and curate sources only
nlm-orch run <query> [flags]       # full pipeline
nlm-orch doctor                    # check tool paths and auth state
nlm-orch --help
```

### Key flags (both `sources` and `run`)
| Flag | Default | Description |
|---|---|---|
| `--config` | none | Path to config file. Wins over all other flags. |
| `--max-results` | 50 | Max YouTube results to fetch before curation. |
| `--recency` | 6months | Recency window (e.g., `6months`, `1year`, `all`). |
| `--max-duration` | 30m | Max video duration. |
| `--min-views` | 1000 | Min view count. |
| `--channel-allow` | none | Comma-separated channel allowlist. |
| `--channel-block` | none | Comma-separated channel blocklist. |

### Key flags (`run` only)
| Flag | Default | Description |
|---|---|---|
| `--notebook-id` | none | Reuse an existing NotebookLM notebook by ID. No reuse-by-name in v1. Default: create new notebook. |
| `--deliverables` | `slides infographic briefing` | Space-separated list of deliverables to generate. Valid values: `slides`, `infographic`, `briefing`. |

## Run ID
Format: `<slug>_<YYYYMMDD_HHMMSS>`

Slug rules (applied in order):
1. Lowercase the query string.
2. Replace any character that is not `[a-z0-9]` with `_`.
3. Collapse consecutive underscores into one.
4. Strip leading and trailing underscores.
5. Truncate to 40 characters.

Timestamp: local time, format `YYYYMMDD_HHMMSS`.

Example: `microbiome_cancer_20260304_143022`

## Output contract (locked)
```
outputs/<run_id>/
  raw.jsonl              # raw yt-dlp metadata for all fetched videos
  sources.json           # final curated list actually uploaded
  run_manifest.json      # full provenance record
  run.log                # stderr + stdout of the run
  artifacts/             # always created, even on --dry-run (may be empty)
    deck.pdf             # present if "slides" in --deliverables  ("slides" -> deck.pdf)
    infographic.png      # present if "infographic" in --deliverables
    briefing.md          # present if "briefing" in --deliverables

Deliverable keyword-to-filename map:
  slides      -> artifacts/deck.pdf
  infographic -> artifacts/infographic.png
  briefing    -> artifacts/briefing.md
```

`artifacts/` is created at run-dir initialization and is present in all cases, including `--dry-run`.

Tracked template: `docs/outputs_template/`

Rules:
- Append-only. Never overwrite prior runs.
- `outputs/` is gitignored.

## Provenance contract (locked)

`sources.json` fields (per source):
```json
{
  "type": "youtube",
  "url": "https://...",
  "title": "...",
  "channel": "...",
  "published_at": "YYYY-MM-DD",
  "duration_seconds": 0,
  "view_count": 0,
  "included": true,
  "exclusion_reason": null
}
```

`run_manifest.json` fields:
```json
{
  "run_id": "...",
  "query": "...",
  "config_path": null,
  "filters": {},
  "notebook_id": "...",
  "notebook_name": "...",
  "prompts_used": [],
  "artifacts": [],
  "missing_artifacts": [],
  "warnings": [],
  "started_at": "ISO8601",
  "finished_at": "ISO8601",
  "status": "success | partial | failed | dry-run",
  "failed_step": null,
  "error_summary": null
}
```

`warnings` is an array of objects for non-fatal issues that did not cause `partial` status.
Each entry has a `type` field (e.g., `"source_add_failed"`, `"prompt_failed"`) and relevant metadata.

---

## Failure handling and status semantics

`status` values:
- `success` : notebook created/selected, at least 1 source added, all requested deliverables downloaded.
- `partial` : run completed but one of the following hard conditions failed:
  - notebook could not be created or selected, OR
  - zero sources were added successfully, OR
  - at least one requested deliverable is missing or failed to download.
- `failed` : run aborted early (tool missing, auth failure, yt-dlp error). `failed_step` and `error_summary` are set.
- `dry-run` : run stopped intentionally after source curation, no NLM calls made.

Non-fatal issues that do NOT trigger `partial`:
- Some (not all) source add operations fail. Recorded in `warnings[]` as `type: source_add_failed` with count and URLs. `sources_failed_urls` also populated for quick reference.
- A prompt ask fails. Recorded in `warnings[]` as `type: prompt_failed`.

On all exit paths after the run folder is created: write (or update) `run_manifest.json`.
Do not delete partial run folders. They are auditable.
Log all errors and warnings to `run.log`.

## Curation defaults (strict)
Applied when no `--config` is given and the flag is not explicitly set:
- Recency: last 6 months
- Max duration: 30 minutes
- Min views: 1000
- Channel allow/block: none

All defaults are overridable by flags or config.

## Decision log
- 2026-03-04: CLI-first design; Claude skills call the CLI to keep runs reproducible.
- 2026-03-04: Append-only outputs with run manifest for provenance.
- 2026-03-04 (interview): Notebook policy = new per run by default; `--notebook-id` for reuse by ID only. No reuse-by-name, no topic lookup table in v1.
- 2026-03-04 (interview): Run ID = slug + timestamp. Slug sanitized from query string.
- 2026-03-05 (update): Run ID format locked to `<slug>_<YYYYMMDD_HHMMSS>`. Slug rules: lowercase, alnum+underscore only, collapse repeats, strip edges, trim to 40 chars.
- 2026-03-04 (interview): Curation defaults = strict (6 months, 30 min, 1000 views).
- 2026-03-04 (interview): Failure handling = leave partial folder, write status=partial manifest.
- 2026-03-04 (interview): Input = positional query string; --config overrides everything.
- 2026-03-04 (interview): Deliverables = all three by default; --no-* flags to skip (superseded).
- 2026-03-05 (update): Deliverables flag changed to single `--deliverables` list. Default: `slides infographic briefing`. Replaces three --no-* flags.
- 2026-03-05 (040_youtube interview): Added `type` field to sources.json schema. Value is `"youtube"` for YouTube sources; future source types will use different values.
- 2026-03-05 (patch): Status semantics locked. `partial` only for notebook failure, zero sources added, or missing deliverable. Source add failures and prompt failures go into `warnings[]`, not partial. `artifacts/` dir created at run-dir init, present even on --dry-run.
